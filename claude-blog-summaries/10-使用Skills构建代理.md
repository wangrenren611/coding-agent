# 使用 Skills 构建代理：为专业化工作装备代理

**发布日期**：2026年1月22日  
**分类**：代理  
**产品**：Claude Code  
**阅读时间**：5 分钟

## 概述

Skills 将域专业知识打包在代理可以访问并应用的文件中——将通用目的代理转变为拥有真实工作能力的知识专家。

## 从域专用代理到 Skills

我们过去认为不同域的代理看起来会非常不同。一个编码代理、一个研究代理、一个用于财务——每个似乎都需要自己的工具和脚手架。该行业最初拥抱这种域专用代理模型。

但随着模型在智能和代理能力方面取得进展，我们收敛于不同的方法。

## 代码即接口

我们看到代码越来越多地被视为几乎所有数字工作的界面，而不仅仅是编程的一个使用案例。Claude Code 是一个编码代理，但也是一个可以通过代码工作的通用目的代理。

考虑使用 Claude Code 生成财务报告：
- 调用研究 API
- 在文件系统中存储数据
- 使用 Python 进行分析
- 综合洞察

所有这些都通过代码发生。脚手架变得就像 bash 和文件系统一样简单。

但通用能力不等于专业知识。当 Anthropic 开始使用 Claude Code 进行真实工作时，出现了一个差距。

## 缺失的环节：域专业知识

谁会希望报税：一个数学天才从第一原理中计算出来，还是一个已经提交了数千份报税的资深税务专业人员？大多数人会选择税务专业人士。并不是因为他们更聪明，而是因为他们有正确的专业知识。

代理今天就像那个数学天才：在通过新颖情况表现出色的推理，但往往缺乏资深专业人士积累的领域专业知识。它们可以做出令人惊叹的事情，只要得到适当的指导。然而，它们经常：
- 错过重要的上下文
- 无法轻易吸收你的组织专业知识
- 无法从重复任务中自动学习

Skills 通过将域专业知识打包在代理可以逐步访问和应用的格式中弥合了这一差距。

## 什么是 Agent Skills？

Skills 将域专业知识和程序知识打包在代理可以访问和应用的文件中。Skills 被组织为目录结构的简单文件集合：

```javascript
anthropic_brand/
├── SKILL.md
├── docs.md
├── slide-decks.md
└── apply_template.py
```

## Skills 的核心特性

### 1. 简单性
文件是一个通用原语，适用于你已经拥有的任何东西。你可以：
- 使用 Git 版本化
- 将其存储在 Google Drive 中
- 与你的团队分享
- 这意味着技能创建并不仅限于工程师。产品经理、分析师和领域专家已经在构建技能以规范他们的工作流程。

### 2. 渐进式披露
为了保护上下文窗口并使技能可组合，它们使用渐进式披露：
- 在运行时，仅显示来自 YAML 前端数据（名称和描述）
- 对于额外细节，技能可以包含一个引用/仅按需加载的支持文档目录

这三级方法意味着你可以在不压倒上下文窗口（元数据使用约 50 个代币）的情况下用数百个技能装备代理——完整 SKILL.md 文件约 500 个代币、引用文件 2,000+ 个代币，并且仅在专门需要时。

### 3. 技能作为工具
传统工具有问题：
- 某些编写得不好，模型无法始终修改或扩展它们
- 它们经常占用上下文窗口
- 它们经常使上下文窗口膨胀

代码是自文档化、可修改的，并且在上下文中不需要一直存在。因此代码在工具集成方面更加优越。

## 技能生态系统

随着技能标准成为代理交互专用能力的方式，一个快速发展的生态系统已经出现：

### 基础技能
这些提供每个人需要的核心能力：处理文档、电子表格、演示文稿等。它们编码文档生成和操作的最佳实践。你可以通过探索[公共仓库中的基础技能](https://github.com/anthropics/skills/tree/main/skills/public)看到其实践中的样貌。

### 合作伙伴技能
随着技能成为标准化代理如何与专用能力交互的方式，公司正在构建技能以使其服务可通过代理访问，同时保持技能格式的简单性。以下是一些例子：
- **K-Dense**：用于基因组测序和单细胞 RNA 分析的工作流
- **Browserbase**：在大型数据库中搜索的代理
- **Notion**：用于协作和知识管理的技能
- 还有许多其他公司的技能

### 企业技能
组织构建打包其内部流程和领域知识的技能，使代理在企业工作中有用。

## Skills 和 MCP 的结合

[Skills 和 MCP 服务器自然地协同工作](https://claude.com/blog/extending-claude-capabilities-with-skills-mcp-servers)。竞争性分析技能可能会协调网络搜索、内部数据库、Slack 消息历史和 Notion 页面，以综合一份全面的报告。

## 新趋势

### 增加复杂性
早期的技能是简单的文档引用。现在我们看到复杂的多步骤工作流，其协调：
- 数据检索
- 复杂计算
- 格式化输出
- 跨多个工具

### Skills 和 MCP
- 技能自然地与 MCP 服务器协同工作
- 用于竞争性分析的专业技能

### 非开发人员采用
技能创建正在扩展到工程师之外，覆盖产品经理、分析师和跨学科的领域专家。他们可以在不到 30 分钟内使用 skill-creator 工具创建和测试他们的第一个技能。

## 完整架构

将所有这些放在一起，新兴的代理架构看起来像以下组合：

1. **代理循环**：核心推理系统，决定下一步做什么
2. **代理运行时**：执行环境（代码、文件系统）
3. **MCP 服务器**：到外部工具和数据源的连接
4. **技能库**：域专业知识和程序知识

每一层都有明确的用途，使系统全面且可扩展。

## 将技能部署到新领域

### 金融服务
就在技能启动后不久，我们增强了 Claude 的金融服务：
- **DCF 模型构建器**：使用适当的 WACC 计算和敏感性分析构建贴现现金流模型
- **可比公司分析**：生成具有相关倍数和基准化的可比表
- **收益分析**：处理季度结果并创建投资更新报告
- **尽职调查**：使用标准化框架构建 M&A 分析
- **推介材料**：遵循行业标准创建客户演示文稿

### 医疗保健与生命科学
我们也通过技能增强了医疗保健和生命科学产品：
- **生物信息学捆绑包**：用于 SCVI 工具和 Nextflow 部署的技能
- **临床试验方案生成**：加速临床试验协议开发用于临床研究
- **科学问题选择**：帮助研究人员识别并构建具有影响力的研究问题
- **FHIR 开发**：帮助开发人员编写更准确的代码用于健康数据互操作性
- **预先授权审查**：减少行政负担并通过交叉引用覆盖要求、临床指南和患者记录加速患者获得所需护理

## 向前推进

我们正在收敛于通用代理、配备 MCP 服务器和技能的架构。真正的价值来自于我们共同构建的集体知识库：捕获专业知识、跨团队转移，并使每个代理比过去更有能力。

## 总结

Skills 代表了一种将域专业知识打包到代理可以访问的文件中的新范式。与构建域专用代理相比，Skills：
- 更简单（基于文件）
- 更易访问（无需编写代码）
- 更灵活（可组合和渐进披露）
- 可共享（可跨团队和生态系统部署）

这使组织能够将其特定领域的专业知识注入到代理中，将通用目的代理转变为高效、专业的助手。
