# Anthropic 增长营销团队如何使用 Claude Code 将广告创作时间从 30 分钟缩短到 30 秒

**发布日期**：2026年1月26日  
**分类**：企业 AI  
**阅读时间**：5 分钟

## 背景

Anthropic 增长营销团队的 Austin Lau 分享了他如何从从未打开终端（terminal）转变为构建 Figma 插件和自动化广告生成工作流——而且没有编写一行代码的故事。

## 从困惑到精通

Austin Lau 最初对 Claude Code 的反应："我不知道这个产品是做什么。作为营销人员，它并没有真正让我产生共鸣。"

但好奇心获胜了。一位 Anthropic 同事在公司 Slack 工作区发布了一份非技术员工如何安装 Claude Code 的指南。Austin 决定遵循其指导。

一周后，Austin 构建了两个从根本上改变他工作方式的工作流：
1. 一个 Figma 插件，可以一键生成广告创意变体
2. 一个 Google Ads 副本生成工作流，帮助他头脑风暴和优化广告文案，然后导出为可上传的 CSV 文件

过去每条广告需要 30 分钟；现在只需 30 秒。

## 问题：淹没在文案和创意变体中

大规模运行绩效营销需要持续生成新鲜的创意：
- Google 响应式搜索广告单独需要 15 个独特的标题
- 每隔几周，文案需要刷新——同时保持品牌声音和传达价值主张

旧流程：
1. 打开 Google Sheets，头脑风暴标题和描述
2. 手动检查字数
3. 复制粘贴到 Google Ads
4. 重复

视觉广告流程更加繁琐：在 Figma 中打开现有框架、多次复制、切换到 Google Doc 抓取文案、切换回 Figma 粘贴文案——横跨十个或更多变体和多个纵横比。

## 解决方案：两个工作流

### Figma 插件用于创意变体
创建广告变体意味着无休止的复制、粘贴和 Figma 与 Google Docs 之间切换。

Austin 使用 Claude Code 构建了一个 Figma 插件：
- 构建时间：约 45 分钟到 1 小时
- 现在节省时间：每次更新大型批量创意时节省近 30 分钟
- 用法：指定创意的框架，然后只需一次复制粘贴所有不同变体和要更新的文案，Figma 插件将创建该单一图像的所有不同排列组合

### Google Ads 副本生成
对于响应式搜索广告，Austin 构建了一个工作流，帮助头脑风暴并创建可上传的广告文案。

工作流程：
1. 输入 `/rsa`（自定义斜杠命令）
2. Claude Code 询问营销数据、现有文案和关键词
3. 交叉参考输入与 Austin 为 Anthropic 创建的 Agent Skills：
   - 品牌声音和语调
   - 产品准确性
   - Google Ads RSA 最佳实践

但输出只是一个起点。"Claude 是一个很棒的头脑风暴伙伴，但有时无法在第一次尝试时获得正确答案。"

在基于 Austin 已知与受众产生共鸣的内容评估每个标题之前，Claude Code 将所有内容组合成一个 CSV 文件，准备上传。

这个工作流建立在人类基础之上。所有文案和示例都是产品营销和文案撰写团队与 Claude 合作编写的。这一强大的起点意味着已经内置了人类判断，甚至在 Claude 开始头脑风暴之前。

## 最佳实践

### 从小步骤开始
"我的建议是：如果你想尝试像 Claude Code 这样的工具，只需思考工作流程中你已确定的那些重复的或可以自动化的领域，然后从一个非常小、简单的事情开始。"

例如，Austin 的第一个项目是为自己构建一个简单的计算器应用，看看 Claude 如何工作。

### 让好奇心驱动工作
Austin 将自己描述为"那种如果我对某件事非常好奇，它几乎会达到固执的程度，我必须找到答案。这种坚持性将他从不知道如何交互的空白终端变成了他日常使用的功能工具。

### 像向同事解释问题一样与 Claude 对话
"你不需要知道如何编写代码。你只需要知道如何清晰、简洁地解释你的挑战和你试图解决的事情。"

非技术员工已经是各自领域中的主题专家。他们了解自己的工作流、感受摩擦点，并且确切知道他们需要从输出中获得什么。Claude Code 为他们提供了自己解决任何重复技术问题的能力。

### 建立在现有资源之上
对于 Figma 插件，Austin 将 Claude Code 指向现有的 Figma API 文档。使用现有文档，Claude 进行了自己的研究并构建了一个原型供 Austin 实验。

一个几乎不起作用的原型足以证明 Austin 头脑中构想的概念，最终导向了功能性产品。

### 更宏大的图景
Austin 认为增长营销人员的角色正在迅速演变。"我认为增长营销正变得几乎像产品经理一样。我们不仅能够执行营销活动，我们还能够实际构建产品以帮助我们实现目标。"

现在他已将工作流向同事和客户演示，他们都有相同的反应：为什么我们没有*更早*这样做？
