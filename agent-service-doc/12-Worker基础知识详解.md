## 1. 什么是 Worker？

### 1.1 定义

**Worker**（工作进程）是后台任务系统的执行单元，负责从任务队列中获取任务并实际执行。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              Worker 本质                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  Worker = 持续运行的进程，持续等待任务，执行任务                             │
│                                                                          │
│  特点:                                                                   │
│  • 独立运行，不依赖客户端连接                                              │
│  • 持续从队列获取任务                                                    │
│  • 执行完成后继续等待下一个任务                                           │
│  • 可以同时运行多个                                                      │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 类比理解

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                            生活类比                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  把 Worker 想象成一个 24 小时快递员:                                       │
│                                                                          │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │                                                                      │  │
│  │   任务队列 = 快递公司的待派送包裹队列                                 │  │
│  │                                                                      │  │
│  │   Worker = 快递员                                                   │  │
│  │   • 一直在店里等待 (启动后持续运行)                                  │  │
│  │   • 从队列取包裹 (从队列获取任务)                                   │  │
│  │   • 派送包裹 (执行任务)                                             │  │
│  │   • 送完回来继续等待 (继续获取下一个任务)                           │  │
│  │                                                                      │  │
│  │   多个 Worker = 多个快递员同时派送                                   │  │
│  │                                                                      │  │
```

## 2. Worker 工作原理

### 2.1 核心流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          Worker 核心工作流程                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│   启动 Worker                                                             │
│        │                                                                 │
│        ▼                                                                 │
│   ┌─────────────────┐                                                     │
│   │ while (true) { │  ← 无限循环，永不退出                             │
│   └────────┬────────┘                                                     │
│            │                                                              │
│            ▼                                                              │
│   ┌─────────────────┐                                                     │
│   │ queue.dequeue() │  ← 阻塞等待，有任务才返回                          │
│   └────────┬────────┘                                                     │
│            │                                                              │
│     ┌──────┴──────┐                                                      │
│     │              │                                                      │
│  有任务          无任务                                                  │
│     │              │                                                      │
│     ▼              │                                                      │
│  ┌──────────────────────┐                                                │
│  │ 1. 获取任务数据       │                                                │
│  │ 2. 更新状态为 running │                                               │
│  │ 3. 执行任务逻辑       │                                                │
│  │ 4. 保存结果          │                                                │
│  │ 5. 更新状态为 completed │                                             │
│  └──────────┬───────────┘                                                │
│             │                                                            │
│     (继续循环，等待下一个任务)                                             │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 代码示例

```typescript
// 最简单的 Worker 实现
class Worker {
  private running = true;
  private queue: TaskQueue;
  
  async start() {
    console.log('Worker 启动');
    
    // 核心：无限循环
    while (this.running) {
      try {
        // 从队列获取任务（阻塞等待）
        const task = await this.queue.dequeue(30);  // 30秒超时
        
        if (task) {
          console.log(`开始执行任务: ${task.taskId}`);
          await this.executeTask(task);  // 执行任务
          console.log(`任务完成: ${task.taskId}`);
        }
        // 如果 timeout，继续循环
        
      } catch (error) {
        console.error('执行出错:', error);
        // 出错不退出，继续等待下一个任务
      }
    }
  }
  
  async executeTask(task: Task) {
    // 执行逻辑
    const result = await doWork(task.payload);
    
    // 保存结果
    await this.saveResult(task.taskId, result);
  }
}

// 启动
const worker = new Worker();
worker.start();
```

## 3. Worker 与任务队列的关系

### 3.1 生产者-消费者模式

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        生产者-消费者模式                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│   ┌──────────────┐          ┌──────────────┐          ┌──────────────┐   │
│   │   生产者     │          │   任务队列    │          │   消费者     │   │
│   │  (API Server) │ ──────▶ │   (Redis)     │ ──────▶ │   (Worker)   │   │
│   │              │          │               │          │              │   │
│   │  提交任务    │          │  存储任务     │          │  获取任务    │   │
│   │              │          │               │          │  执行任务    │   │
│   └──────────────┘          └──────────────┘          └──────────────┘   │
│                                                                          │
│   时序:                                                                   │
│   1. API 提交任务 → 加入队列                                             │
│   2. Worker 从队列取出任务                                                │
│   3. Worker 执行任务                                                      │
│   4. Worker 继续等待下一个任务                                            │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3.2 队列操作详解

```typescript
// 任务队列核心操作

class TaskQueue {
  private redis: Redis;
  
  /**
   * 生产者：提交任务
   * 
   * LPUSH: 从左边插入
   * BRPOP: 从右边取出（阻塞等待）
   */
  async enqueue(task: TaskMessage): Promise<void> {
    // 任务加入队列（左进）
    await this.redis.lpush('task:queue', JSON.stringify(task));
  }
  
  /**
   * 消费者：获取任务
   * 
   * BRPOP 特性：
   * - 如果队列有任务，立即返回
   * - 如果队列为空，阻塞等待
   * - timeout=0 表示无限等待
   */
  async dequeue(timeout: number = 0): Promise<TaskMessage | null> {
    const result = await this.redis.brpop('task:queue', timeout);
    
    if (!result) return null;
    
    const [, data] = result;
    return JSON.parse(data);
  }
}
```

### 4.2 启动代码

```typescript
// Worker 启动脚本
async function main() {
  // 1. 创建 Worker 实例
  const worker = new Worker();
  
  // 2. 处理进程信号（优雅停止）
  process.on('SIGTERM', async () => {
    console.log('收到 SIGTERM，正在停止...');
    await worker.stop();
    process.exit(0);
  });
  
  process.on('SIGINT', async () => {
    console.log('收到 SIGINT，正在停止...');
    await worker.stop();
    process.exit(0);
  });
  
  // 3. 启动 Worker
  await worker.start();
}

// 运行
main().catch(error => {
  console.error('Worker 启动失败:', error);
  process.exit(1);
});
```

```
┌─────────────────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────────────────┤

│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │  Worker 进程 1          Worker 进程 2          Worker 进程 3      │  │

│  │  │  (1 核)     │        │  (1 核)     │        │  (1 核)     │    │  │
│  │  └─────────────┘        └─────────────┘        └─────────────┘    │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
│                                                                          │
│  优点:                                                                   │
│  • 简单稳定，一个任务崩溃不影响其他                                       │

│  • 易于调试                                                              │
│                                                                          │
```

## 6. Worker 扩缩容

### 6.1 为什么需要扩缩容？

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           扩缩容原因                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  场景 1: 流量高峰期                                                     │
│  ──────────────────────────────                                           │
│  • 早上 9 点到下午 6 点，用户量大幅增加                                  │
│  • 任务队列积压越来越多                                                   │
│  • 需要更多 Worker 处理任务                                               │
│                                                                          │
│  场景 2: 流量低谷期                                                      │
│  ──────────────────────────────                                           │
│  • 凌晨到早上，用户量很少                                                 │
│  • 任务很少，大部分 Worker 空闲                                           │
│  • 减少 Worker，节省成本                                                 │
│                                                                          │
│  场景 3: 突发流量                                                        │
│  ──────────────────────────────                                           │
│  • 促销活动，瞬间大量用户                                                  │
│  • 队列积压急剧增加                                                      │
│  • 快速扩容应对                                                           │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 6.2 手动扩缩容

```bash
# PM2 手动扩缩容

# 查看当前 Worker 数量
pm2 list

# 扩容：增加 5 个 Worker
pm2 scale agent-worker +5

# 缩容：减少到 3 个 Worker
pm2 scale agent-worker 3

# 查看实时日志
pm2 logs agent-worker
```

### 6.3 自动扩缩容 (HPA)

```yaml
# Kubernetes HorizontalPodAutoscaler
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: worker-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: agent-worker
  minReplicas: 10    # 最小 10 个
  maxReplicas: 100   # 最大 100 个
  metrics:
    # 队列长度每超过 100/Worker 就扩容
    - type: External
      external:
        metric:
          name: queue_length
        target:
          type: AverageValue
          averageValue: "100"
  behavior:
    scaleUp:
      stabilizationWindowSeconds: 60
      policies:
        - type: Percent
          value: 100   # 每次翻倍
          periodSeconds: 60
    scaleDown:
      stabilizationWindowSeconds: 300  # 缩容冷却 5 分钟
```


### 7.2 健康检查

```typescript
// Worker 健康检查
class WorkerHealthCheck {
  async check(): Promise<HealthStatus> {
    const checks = {
      // 检查 Redis 连接
      redis: await this.checkRedis(),
      
      // 检查数据库连接
      database: await this.checkDatabase(),
      
      // 检查队列是否可访问
      queue: await this.checkQueue(),
      
      // 检查内存使用
      memory: this.checkMemory(),
    };
    
    const allHealthy = Object.values(checks).every(c => c.healthy);
    
    return {
      status: allHealthy ? 'healthy' : 'unhealthy',
      checks,
    };
  }
  
  private checkMemory(): HealthCheckResult {
    const used = process.memoryUsage();
    const heapUsedMB = used.heapUsed / 1024 / 1024;
    const heapTotalMB = used.heapTotal / 1024 / 1024;
    const usagePercent = (heapUsedMB / heapTotalMB) * 100;
    
    return {
      healthy: usagePercent < 80,
      message: `Heap: ${heapUsedMB.toFixed(0)}MB / ${heapTotalMB.toFixed(0)}MB`,
    };
  }
}
```

### 8.2 故障恢复机制

```typescript
// 故障恢复机制

class WorkerRecovery {
  /**
   * 1. 任务超时处理
   */
  async handleTimeout(task: Task, worker: Worker): Promise<void> {
    // 强制终止任务
    await worker.terminateTask(task.id);
    
    // 更新任务状态
    await this.taskStore.update(task.id, {
      status: 'failed',
      error: 'Task execution timeout',
    });
    
    // 记录告警
    await this.alert.send({
      type: 'task_timeout',
      taskId: task.id,
      duration: task.timeout,
    });
  }
  
  /**
   * 2. Worker 崩溃检测
   */
  async detectCrashedWorkers(): Promise<void> {
    const workers = await this.workerRegistry.list();
    const now = Date.now();
    
    for (const worker of workers) {
      // 超过 2 分钟没有心跳，认为崩溃
      if (now - worker.lastHeartbeat > 120000) {
        await this.handleWorkerCrash(worker);
      }
    }
  }
  
  /**
   * 3. 处理 Worker 崩溃
   */
  private async handleWorkerCrash(worker: Worker): Promise<void> {
    // 查找该 Worker 正在执行的任务
    const runningTasks = await this.taskStore.findByWorkerId(worker.id);
    
    // 将任务标记为待恢复
    for (const task of runningTasks) {
      await this.taskStore.update(task.id, {
        status: 'pending',  // 重新入队
        workerId: null,     // 清除崩溃的 Worker
        error: 'Worker crashed, task will be retried',
      });
      
      // 重新入队
      await this.queue.enqueue(task);
    }
    
    // 记录告警
    await this.alert.send({
      type: 'worker_crash',
      workerId: worker.id,
      tasksAffected: runningTasks.length,
    });
  }
}
```
